# 规则运行功能 - 快速启动指南

## 🚀 立即开始使用

### 步骤1: 确保Flask应用正在运行

如果应用未运行，启动它：
```bash
cd d:\work\dataquality\dataq\platform
python src/backend/app.py
```

应用将在 `http://localhost:5000` 启动。

---

### 步骤2: 访问运行管理页面

在浏览器中打开：
```
http://localhost:5000/execution-management
```

你将看到：
- **顶部**: "批量运行选中资产" 和 "运行所有激活资产" 按钮
- **中间**: 资产执行列表（可多选）
- **底部**: 最近执行记录

---

### 步骤3: 测试手动运行

#### 方式A: 运行单个资产
1. 找到任意一个有规则的资产
2. 点击该行的 "▶️ 运行" 按钮
3. 确认对话框中点击"确定"
4. 查看执行结果弹窗

#### 方式B: 批量运行
1. 勾选多个资产的复选框
2. 点击顶部的 "▶️ 批量运行选中资产" 按钮
3. 确认对话框中点击"确定"
4. 等待所有资产执行完成
5. 查看批量运行结果汇总

---

### 步骤4: 配置定时调度

1. 找到要调度的资产
2. 点击 "⏰ 调度" 按钮
3. 在弹出的对话框中输入调度类型：
   - 输入 `interval` 表示间隔调度
   - 或输入 `cron` 表示Cron表达式调度
4. 根据提示输入参数：
   - **间隔调度**: 输入小时数（如 `24`）
   - **Cron调度**: 输入表达式（如 `0 9 * * *` 表示每天9点）
5. 确认配置成功

**验证调度是否生效**:
```bash
curl http://localhost:5000/api/v1/assets/1/schedule/status
```

应该返回调度信息。

---

### 步骤5: 查看执行历史

#### 在运行管理页面
- 页面下方显示"最近执行记录"表格
- 包含：执行ID、资产名称、触发方式、成功率等
- 点击"详情"查看完整信息

#### 在校验历史页面
```
http://localhost:5000/validations
```
- 查看所有历史记录
- 按时间倒序排列
- 点击"详情"跳转到详情页

---

### 步骤6: 创建问题工单

当校验失败时：

#### 自动创建
如果在配置调度时设置了 `auto_create_issue=true`，系统会自动为失败的规则创建问题工单。

#### 手动创建
1. 查看执行结果（在执行结果弹窗中）
2. 如果有失败规则，会显示 "📝 创建问题工单" 按钮
3. 点击按钮跳转到问题管理页面
4. 表单已预填充校验信息
5. 补充详细信息后提交

---

## 🔍 故障排查

### 问题1: 点击"运行"按钮没有反应

**可能原因**:
- 浏览器控制台有JavaScript错误
- 后端API未正常响应

**解决方法**:
1. 打开浏览器开发者工具（F12）
2. 切换到Console标签
3. 查看是否有错误信息
4. 检查Network标签中的API请求状态

---

### 问题2: 执行结果显示"文件不存在"

**可能原因**:
- 资产的数据源路径不正确
- CSV/Excel文件已被移动或删除

**解决方法**:
1. 进入资产管理页面
2. 编辑该资产
3. 确认数据源路径正确
4. 重新上传数据文件（如果需要）

---

### 问题3: 调度配置后没有自动执行

**可能原因**:
- Flask应用未启动调度器
- Cron表达式格式错误
- 资产没有激活的规则

**解决方法**:
1. 检查Flask应用日志，确认调度器已启动
2. 验证Cron表达式格式（分 时 日 月 周）
3. 确认资产有至少一个激活的规则
4. 检查资产的 `is_active` 状态为 `true`

---

### 问题4: 执行历史中没有trigger_type字段

**可能原因**:
- 数据库迁移脚本未执行

**解决方法**:
```bash
cd d:\work\dataquality\dataq\platform
python migrations/add_trigger_type_to_validation_history.py
```

然后重启Flask应用。

## 💡 最佳实践

### 1. 合理设置调度频率
- **核心业务表**: 每小时或每6小时
- **重要业务表**: 每天1-2次
- **一般业务表**: 每周1-2次

### 2. 启用自动问题创建
对于关键资产，建议配置：
```json
{
  "auto_archive": true,
  "auto_create_issue": true
}
```
这样可以及时发现和处理数据质量问题。

### 3. 定期检查执行历史
- 每周查看一次执行历史
- 关注成功率下降的资产
- 分析失败规则的共性

### 4. 使用标签化管理
虽然当前版本不支持标签，但可以通过资产命名规范来分类：
- `prod_user_table` - 生产环境用户表
- `test_order_data` - 测试环境订单数据
