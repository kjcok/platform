## 🏗️ 模块详细说明

### 1. Web 服务层 (app.py)

**职责**: 
- HTTP 请求路由
- 参数验证
- 响应格式化

**主要接口**:
```python
GET  /                    # 主页
POST /api/upload          # 上传文件
POST /api/evaluate        # 执行评估
GET  /report/<filename>   # 查看报告
```

**代码量**: ~155 行

---

### 2. 文件管理模块 (file_manager.py)

**职责**:
- 文件上传和保存
- 文件格式验证
- 元数据提取（列名）

**核心函数**:
```python
save_uploaded_file(file, upload_folder)  # 保存上传文件
read_file_columns(file_path)             # 读取列名
check_file_exists(file_id, upload_folder) # 检查文件存在性
```

**支持格式**: CSV, XLSX, XLS

**代码量**: ~110 行

---

### 3. GE 评估引擎 (ge_engine.py)

**职责**:
- 规则映射和转换
- 调用 Great Expectations 执行验证
- 结果提取和汇总

**支持的规则类型**:
```python
RULE_MAPPING = {
    'not_null': 'expect_column_values_to_not_be_null',
    'unique': 'expect_column_values_to_be_unique',
    'between': 'expect_column_values_to_be_between',
    'in_set': 'expect_column_values_to_be_in_set',
    'match_regex': 'expect_column_values_to_match_regex',
    'type_string': 'expect_column_values_to_be_of_type',
    ...
}
```

**核心流程**:
1. 读取数据 → Pandas DataFrame
2. 转换为 GE 数据集
3. 动态应用规则
4. 收集验证结果
5. 计算成功率

**代码量**: ~143 行

---

### 4. 报告渲染器 (report_renderer.py)

**职责**:
- 接收评估结果
- 渲染 Jinja2 模板
- 生成 HTML 报告

**核心函数**:
```python
generate_report(result_data, reports_folder)  # 生成报告
get_rule_name(rule_type)                       # 规则名称映射
```

**报告内容**:
- 总体摘要卡片
- 详细结果表格
- 异常值示例
- 可视化进度条

**代码量**: ~60 行

---

### 5. 前端页面 (templates/)

#### index.html - 主页面
**功能**:
- 文件上传界面
- 动态规则配置
- AJAX 异步请求
- 响应式布局

**技术栈**:
- HTML5 + CSS3
- jQuery 3.6.0
- 原生 JavaScript

**代码量**: ~424 行

#### report_template.html - 报告模板
**功能**:
- 展示评估结果
- 可视化图表
- 打印友好样式

**特性**:
- 渐变色设计
- 响应式布局
- 状态徽章（通过/失败）

**代码量**: ~310 行

## 🎯 设计原则

### 1. 极简主义
- **目标**: 核心代码 < 500 行
- **实现**: 
  - 不使用复杂的配置文件
  - 不使用数据库
  - 不使用消息队列
  - 同步处理，无异步复杂性

### 2. 模块化
- 清晰的模块边界
- 单一职责原则
- 易于测试和维护

### 3. 可扩展性
- 规则映射字典 → 轻松添加新规则
- 插件化架构 → 可替换评估引擎
- RESTful API → 易于集成

### 4. 零配置
- 无需环境变量
- 无需外部服务
- `pip install` + `python app.py` 即可运行

## 📝 总结

本平台的架构设计遵循以下核心理念：

1. **简单优于复杂** - 用最少的代码实现核心功能
2. **约定优于配置** - 零配置启动，开箱即用
3. **组合优于继承** - 模块化设计，灵活组合
