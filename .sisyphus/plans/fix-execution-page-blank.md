# 修复执行管理页面空白问题

## 问题诊断

执行管理页面空白的根因：**资产列表加载时的串行阻塞**
- `loadAssets()` 函数对每个资产串行调用 2 个 API
- 每个 API 都是 `await`，导致整个资产列表渲染被阻塞
- 如果其中一个 API 延迟或失败，整个页面保持空白

## 任务清单

### Task 1: 优化 loadAssets 函数 - 先渲染再异步加载
- **文件**: `src/frontend/static/js/execution_management.js`
- **修改**: 第 15-31 行 `loadAssets()` 函数
- **内容**:
  1. 获取资产列表后立即调用 `renderAssetList()` 显示基础数据
  2. 将串行 `await` 改为 `Promise.all` 并行加载附加信息
  3. 附加信息加载完成后再更新列表显示

### Task 2: 验证修复效果
- 访问 `/execution-management` 页面
- 确认资产列表在 1 秒内显示（不再空白）
- 确认调度状态和最后执行时间稍后异步更新
- 检查浏览器控制台无错误

## 预期效果

- ✅ 页面打开后立即显示资产列表（不再长时间空白）
- ✅ 调度状态和最后执行时间稍后异步更新
- ✅ 单个 API 失败不影响整个列表的显示
- ✅ 最近执行记录正常显示
